# AutoPilot Code Team — 视频讲解PPT大纲

> 从"有想法却不会编程"的普通人，到"用AI团队做Vibe产品"的实战者

---

## 第1页：痛点开场（5秒）

**标题**：你有一个想法，但不知道怎么做出来

**要点**：
- 有想法：想做一个工具、一个产品、一个应用
- 但不会写代码：HTML、JavaScript、Python，这些词听着就头大
- 尝试过：买了编程课、看了教程，一做就卡壳
- 找了人：外包太贵，朋友没空，需求说不清楚

**画面建议**：屏幕中间一个大问号，周围飘着"有想法""不会编程""外包贵""需求说不清"等关键词，节奏快闪，配焦虑感BGM

## 第2页：AI来了，但问题还在（10秒）

**标题**：AI能写代码了，但你不等于会编程了

**要点**：
- Cursor、Claude Code、Kimi Code 能写代码，但给谁用？
- 你问AI："帮我做个App"，AI问你："你要什么功能？用什么技术？数据库怎么设计？"
- 你愣住了："我不知道，我要是知道还用你？"
- 市场上大量的"编程Skill"、"Prompt教程"，写的全是给工程师看的，不是给你看的
- 你手里只有一把Cursor，但没有一张地图

**画面建议**：左边是用户迷茫的脸，右边是AI工具界面，中间是一个巨大的"认知鸿沟"裂缝，裂缝下面写着"从想法到产品，缺的不是工具，是路径"

## 第3页：市场上的Skill，为什么帮不到你（15秒）

**标题**：为什么你看了100个编程教程，还是不会做

**要点**：
- 市面上的教程和Skill，假设你已经是半个工程师
- 它们教你："写一个React组件""配置Docker""设计数据库Schema"
- 你真正需要的是："我现在该做什么？下一步该找谁？谁负责什么？"
- 你不需要学会编程，你需要一个团队来帮你编程
- 但真正的团队：太贵、太难找、太麻烦

**画面建议**：分三列对比，左边"你看到的教程"（代码截图、技术术语），中间"你需要的"（清晰分工、操作步骤），右边"你得到的"（困惑、放弃），用箭头连接，强调落差

## 第4页：核心洞察（10秒）

**标题**：Vibe Coding 时代，一个人需要一个团队

**要点**：

**画面建议**：一个人坐在电脑前，周围环绕21个发光的小人图标（产品、设计、架构、前端、后端、DevOps...），每个图标都有名字和职责，像指挥中心一样

## 第5页：改造思路 — 5个设计原则（20秒）

**标题**：这套角色库，为什么对小白有用

**要点**（5个原则，每页一个或一页合并）：

1. **零角色扮演，纯能力定义**
   - 不说"你是10年经验的Alex"，说"本智能体负责原型验证，使用v0、Lovable、Bolt"
   - 你知道它是什么，就能用它做什么

2. **零Emoji，全专业**
   - 没有花哨的表情符号，全是可执行的System Prompt
   - 导入Cursor、Claude Code、Kimi Code就能直接工作

3. **RICE-V 评分，数据驱动**
   - 不只是优先级，还加Vibe Speed（验证速度）和Model Risk（模型风险）
   - 让你知道：这个功能2小时能验证，还是2周？值不值得做？

4. **AI可观测性，每个角色有指标**
   - 幻觉率、TTFT（响应时间）、Token成本、人工接管率
   - 你知道AI做得怎么样，不是盲目信任

5. **可执行Spec，不是静态文档**
   - 每个角色输出的是Prompt模板、代码配置、检查清单
   - AI能直接消费，不是给你看的，是给AI执行的

**画面建议**：5个原则分别用5个简洁的图标+一句话，背景用深色科技感，文字白色高对比，每出现一条原则就配一个简短的动画效果

**标题**：3步，让你从"有想法"到"有产品"

**步骤**：
1. **选角色**：根据你的项目阶段，选一个角色（比如先从 `vibe-prototyper` 原型工程师开始）

**画面建议**：三步分别用三个大数字图标，下方配简单的动画：选角色（点击图标）→ 导入（复制粘贴）→ 对话（输入框打字），每一步有箭头连接，最后出结果画面

## 附录：PPT制作建议

| 项目 | 建议 |
|------|------|
| **配色** | 深海藏蓝（#0f172a）+ 青碧（#3b82f6）+ 白色文字，拒绝花里胡哨 |
| **字体** | 标题用粗体无衬线（Inter/思源黑体），正文用常规体，大小对比强烈 |
| **动画** | 简洁进入动画，不要花哨旋转/弹跳，每页2-3个元素依次出现 |
| **音效** | 科技感的电子BGM，轻节奏，不抢语音 |
| **语音** | 推荐剪映AI配音，"磁性男声"或"新闻女声"，语速1.2倍 |
| **总时长** | 约120秒（2分钟），适合抖音/视频号 |
| **画面比例** | 9:16竖屏（手机端），或16:9横屏（B站/YouTube） |
| **节奏** | 每页5-15秒，快速切换，不拖沓，信息密度高 |
| **不要** | 不要emoji、不要大段文字、不要动画堆砌、不要代码截图（普通人看不懂） |
